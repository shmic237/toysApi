Users-Routes

#Method URL Body Description
GET /api/v1/users none Get all users.
GET /api/v1/users/info/{id} none Get your own info.
POST /api/v1/users/register

{
"firstName": "Michal",
"lastName": "Shlezinger",
"email": "michal@gmail.com",
"password": "michal1234"
}

Add a new user to the data base.
POST /api/v1/users/login

{
"email": "michal@gmail.com",
"password": "michal1234"
}

User login, if the email and password are correct the server will respond error.
DELETE /api/v1/users/{idDelete} Delete a user from the data base.
User can delete only himself.
PATCH /users/{idEdit} Patch a user from the data base.
User can patch only himself.

Toys-Routes

#Method URL Body Description
GET /api/v1/toys none Get a list of all the toys, will present 10 results per page.
GET /api/v1/toys/{id} none Get your own toys.
GET /api/v1/toys/search?s={name/info} none search for a toy by searching follwing name or info after the "s=",
it will present 10 results per page.
GET /api/v1/toys/category/{catname} none Get a list of all the toys category, it will present 10 results per page.
GET /api/v1/toys/single/{id} none Get a toy.
POST /api/v1/toys

{
"name": "toy",
"info": "new toy",
"category": "category toy",
"img_url": "toy.jpg",
"price": 59
}

Add a new toy to the data base.
You need a token for this query
DELETE /api/v1/toys/{idDelete} Delete a toy from the data base.
User can only patch his own toys - need a token for this query.
PATCH /api/v1/toys/{idEdit} Patch a toy from the data base.
User can only patch his own toys - need a token for this query.
